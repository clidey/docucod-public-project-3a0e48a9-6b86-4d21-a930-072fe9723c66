# Quickstart Workflow

Step through the core user journey: define models and query interfaces, run the code generator, and start using your new APIs. Each step is linked to the most relevant source or example for maximum clarity.

---

## 1. Define Your Models

Start by defining your Go structs to represent your database models as usual. These structs will serve two purposes:

- Define the schema and associations for your database entities
- Drive generation of *field helpers* that provide type-safe API for filtering, updating, and working with associations

Example:

```go
// examples/models/user.go
type User struct {
    gorm.Model
    Name  string
    Age   int
    Email string
}
```

**Tip:** Add proper struct tags and consider annotating fields if you want custom field helper mappings (see the [Custom Field Mapping](https://gorm.io/cli/gorm/guides/advanced-usage/custom-field-helpers) page).

---

## 2. Create Query Interfaces

In the same package or directory, write Go interfaces describing your queries. Embed raw SQL templates as method comments using GORM CLI's DSL. This declarative approach allows the generator to build *type-safe query APIs* for you.

Example:

```go
// examples/query.go
type Query[T any] interface {
  // SELECT * FROM @@table WHERE id=@id
  GetByID(id int) (T, error)

  // SELECT * FROM @@table WHERE @@column=@value
  FilterWithColumn(column string, value string) (T, error)

  // UPDATE @@table
  // {{set}}
  //   {{if user.Name != ""}} name=@user.Name, {{end}}
  //   {{if user.Age > 0}} age=@user.Age {{end}}
  // {{end}}
  // WHERE id=@id
  UpdateUser(user T, id int) error
}
```

**Insight:** The placeholders like `@@table`, `@param`, and template DSL blocks such as `{{set}}` support fully customizable SQL while keeping interfaces clean and declarative.

---

## 3. Run the Code Generator

With your models and query interfaces defined, invoke the CLI to generate the corresponding Go code.

Command:

```bash
gorm gen -i ./examples -o ./generated
```

- `-i` points to your input directory containing interfaces and models
- `-o` specifies the output directory for generated type-safe APIs and field helpers

**What Happens:**
- The generator parses your Go files
- Extracts interfaces with SQL templates
- Scans models for fields and associations
- Generates fluent, type-safe API code for queries and updates

---

## 4. Use Your Generated APIs

Now you can leverage the generated code in your application for safe and fluent database operations.

### Example Queries and Updates

```go
import (
    "context"
    "gorm.io/gorm"
    "generated" // your generated package
    "examples/models" // your models package
)

func exampleUsage(db *gorm.DB, ctx context.Context) error {
    // Query by ID
    user, err := generated.Query[models.User](db).GetByID(ctx, 123)
    if err != nil {
        return err
    }

    // Filter by dynamic column
    u, err := generated.Query[models.User](db).FilterWithColumn(ctx, "email", "user@example.com")
    if err != nil {
        return err
    }

    // Update using SQL template
    err = generated.Query[models.User](db).
        UpdateUser(ctx, models.User{Name: "newname", Age: 30}, 123)
    if err != nil {
        return err
    }

    return nil
}
```

### Using Field Helpers

Generated field helpers let you write expressive, type-safe predicates and update setters:

```go
// Find adults
db.Where(
  generated.User.Age.Gt(18),
).Find(ctx, &users)

// Update a user by name
db.Set(
  generated.User.Name.Set("newname"),
).Where(
  generated.User.ID.Eq(123),
).Update(ctx)
```

## Practical Tips & Best Practices

- **Keep interfaces focused:** Only declare methods you need; keep SQL templates clear and simple.
- **Use configuration:** Override output paths and field mappings with `genconfig.Config` in your packages.
- **Version control generated code:** Commit generated code or use a build process to generate it as needed.
- **Reference examples:** Examine `examples/query.go` and `examples/models` for practical interface and model patterns.
