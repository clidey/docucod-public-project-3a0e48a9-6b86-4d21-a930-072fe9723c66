# Troubleshooting Your First Run

This guide helps you resolve common errors encountered when generating code with GORM CLI, such as misconfigured interface models, missing method declarations, or issues with the output directory. By following the practical steps here, ensure your initial code generation experience is smooth and productive.

---

## 1. Common Error Scenarios & How to Fix Them

### 1.1 Misconfigured Input Path or Missing Input Files

**Problem:** You run the generation command but encounter errors like "file not found" or "no interfaces found".

**Cause:** The input path to your interface file or directory is incorrect, does not exist, or contains no valid Go interface definitions with SQL annotations.

**Solution:**
- Verify the exact path to your `.go` files that declare your query interfaces.
- Ensure your interfaces include methods with SQL template comments.
- Use relative or absolute paths correctly in the `-i` flag.

```bash
gorm gen -i ./examples/query.go -o ./generated
```

<Tip>
Always ensure your interface files and models are in the provided input directory or file path. The CLI requires Go interfaces annotated with SQL to generate code correctly.
</Tip>

---

### 1.2 Missing or Incorrect Flags

**Problem:** The generator fails with messages about required flags or unexpected input.

**Cause:** The `--input` (`-i`) flag is missing or invalid.

**Solution:**
- Always specify the input file or directory using `-i`.
- Optionally specify `-o` for the output directory; if omitted, defaults to `./g`.

```bash
gorm gen -i ./path/to/your/interfaces -o ./generated
```

---

### 1.3 Output Directory Permission Issues

**Problem:** Errors occur creating or writing generated files, e.g., permission denied.

**Cause:** The output directory does not exist, or your user does not have write permissions.

**Solution:**
- Manually create the output directory with appropriate permissions.

```bash
mkdir -p ./generated
chmod u+w ./generated
```

- Alternatively, ensure your CLI command is run with sufficient permissions.

<Warning>
Using elevated permissions (like `sudo`) is generally discouraged unless necessary for your environment.
</Warning>

---

### 1.4 Empty or Missing Interface Methods

**Problem:** Generation completes but no code methods are generated.

**Cause:** Your interfaces might lack methods with raw SQL comments, or your `genconfig.Config` filters exclude your interfaces.

**Solution:**
- Ensure your interfaces contain methods documented with proper SQL template comments.
- Check any `genconfig.Config` settings for `IncludeInterfaces` or `ExcludeInterfaces` that may filter out your interfaces.

Example interface with SQL template:

```go
// Query defines user queries
 type Query[T any] interface {
   // SELECT * FROM @@table WHERE id=@id
   GetByID(id int) (T, error)
 }
```

---

### 1.5 Template Syntax or SQL Annotation Errors

**Problem:** Parsing or rendering errors related to SQL templates in method comments.

**Cause:** Incorrect use of the DSL directives (`@@table`, `@param`, `{{where}}`, etc.) or malformed SQL.

**Solution:**
- Review your interface method comment syntax carefully.
- Ensure placeholders match method parameters correctly.
- Use supported directives as documented in the [Template DSL](#).

Example:

```go
// SELECT * FROM @@table WHERE id=@id
GetByID(id int) (T, error)
```

---

### 1.6 Context Parameter Missing in Method Signature

**Problem:** Compiler panics or error indicating wrong method signature during generation.

**Cause:** Your method does not explicitly use a `context.Context` argument.

**Solution:**
- The generator inserts `ctx context.Context` automatically if absent, but ensure method signatures conform to expectations.
- Method return values must include `error` as the last return type.

Example correct signature:

```go
GetByID(ctx context.Context, id int) (T, error)
```

The generator adds `ctx` implicitly if omitted, so both signatures work, but consistency helps avoid confusion.

## 2. Verify Your Environment Setup

Before troubleshooting deeper, confirm:

- Your Go version is 1.18 or newer.
- The GORM CLI binary is installed and accessible from your PATH.

Run:

```bash
gorm version
```

If this command errors, reinstall the CLI using:

```bash
go install gorm.io/cli/gorm@latest
```

## 3. Diagnosing Generation Failures

### Step-by-Step Diagnostic Workflow

<Steps>
<Step title="Step 1: Confirm Input Files">
Check that input files exist and contain interfaces with SQL comments.

```bash
ls -l ./examples/query.go
```

Open and verify interface methods have SQL templates.
</Step>

<Step title="Step 2: Run CLI Verbosely">
Run the CLI with error output captured.

```bash
gorm gen -i ./examples -o ./generated
```

Monitor output for error messages or warnings.
</Step>

<Step title="Step 3: Inspect Generated Files">
Check if any files were created in the output directory.

```bash
ls -l ./generated
```

If empty, examine interface configuration or filters.
</Step>

<Step title="Step 4: Test Build Generated Code">
Build or test your generated code to verify correctness.

```bash
cd ./generated
 go build ./...
```

Fix build errors by reviewing your interface and model definitions.
</Step>

</Steps>

## 4. Practical Tips & Best Practices

- **Always include SQL annotations on interface methods.** The generator depends on these comments to produce query code.
- **Use relative paths consistently** when specifying input and output directories.
- **Avoid excluding interfaces or structs unintentionally** with broad glob patterns in `genconfig.Config`.
- **Check file permissions and directory ownership** for your output folder before running.
- **Run a test with a minimal interface** to isolate issues:

```go
// Minimal example interface for testing
package test

type Query[T any] interface {
  // SELECT 1
  Test() (T, error)
}
```

Then generate and verify success before expanding.

## 5. Troubleshooting Examples

<AccordionGroup title="Example Errors & How to Resolve">
<Accordion title="Error: no interfaces found in input path">
- Check the input path contains interface files
- Verify interfaces have SQL comments
- Check `genconfig.Config` filters do not exclude them
</Accordion>

<Accordion title="Error: failed to create output directory">
- Manually create the output folder
- Check user permissions
- Use a directory where you have write access
</Accordion>

<Accordion title="Error: method return values invalid">
- Ensure methods return either `(T, error)` or `error` as last value
- Avoid more than two return values
- Use pointer receivers and remember context injection
</Accordion>

<Accordion title="Error: template parse failure">
- Confirm usage of supported DSL annotations
- Check SQL syntax correctness
- Watch for accidental escape characters or malformed comments
</Accordion>

</AccordionGroup>
